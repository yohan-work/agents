## 인터뷰 1: minsu_kim (초급 개발자)
**주제**: 온보딩 & 첫인상
**Q**: Antigravity를 처음 사용했을 때 어땠나요?
**A**: 솔직히 좀 혼란스러웠어요. 리가장 설치했는데, API 키를 별도의 JSON 파일에서 설정해야 한다는 걸 몰랐거든요. 채팅창 띄우는 데만 30분 걸렸습니다.
**Q**: 어떤 점이 도움이 되었을까요?
**A**: 설정 마법사요? 아니면 그냥 설정 UI라도요. 설정 때문에 JSON 파일 편집하는 건 너무 90년대 방식 같아요.
**Q**: 작동하고 나서는 어땠나요?
**A**: 꽤 멋졌는데, 가끔 존재하지 않는 import를 만들어내더라고요(할루시네이션).

## 인터뷰 2: sarah_lee (시니어 프론트엔드 엔지니어)
**주제**: 일상적인 워크플로우
**Q**: 업무에서 Antigravity를 어떻게 쓰시나요?
**A**: 주로 보일러플레이트 코드 짤 때 써요. 근데 큰 문제가 하나 있어요. 바로 '컨텍스트 윈도우'예요.
**Q**: 좀 더 자세히 말씀해 주시겠어요?
**A**: 저는 대규모 Next.js 레포에서 일해요. 5개의 다른 컴포넌트를 import하는 컴포넌트를 리팩토링하려고 하면, 다른 파일에 정의된 타입을 자꾸 까먹어요. 제가 명시적으로 파일을 열지 않으면 "그 파일은 볼 수 없습니다"라고 하죠. 컨텍스트를 주려고 파일 10개를 일일이 여는 건 너무 귀찮아요.
**Q**: 더 나은 자동 컨텍스트 기능을 원하시는군요?
**A**: 네! import를 읽어서 해당 파일들을 자동으로 가져올 만큼 똑똑해야죠.

## 인터뷰 4: joon_choi (백엔드 엔지니어)
