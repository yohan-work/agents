# 사용자 인터뷰 인사이트 리포트

**날짜**: 2026-01-14
**출처**: 10건의 가상 인터뷰

## 1. 요약 (Executive Summary)
사용자들은 Antigravity의 지능적인 면을 높이 평가하지만 **컨텍스트 인식 부족**과 **설정의 어려움**으로 인해 고전하고 있습니다. 숙련된 사용자들은 기존 워크플로우(로컬 모델, Vim, 디자인 시스템)와의 더 나은 통합을 요구하는 반면, 초보자들은 설정 과정을 벅차게 느낌니다.

## 2. 주요 발견 (Key Findings)

### A. 컨텍스트 인식 부재 (높은 중요도)
- 다수의 사용자(시니어 개발자, 테크 리드, 디자이너)가 에이전트가 프로젝트별 컨텍스트를 무시한다고 보고했습니다.
- 에이전트가 읽지 못하는 것들:
    - import된 파일 (Next.js 컴포넌트 등)
    - 문서 (`docs/` 폴더)
    - 디자인 토큰 (`tailwind.config.js`)
- **영향**: 수동 수정이 필요한, 문맥에 맞지 않는(할루시네이션) 코드를 생성함.

### B. 온보딩 마찰
- 초보자들은 JSON 기반 설정에 어려움을 겪음.
- "설정 마법사"나 UI 기반 설정 기능의 부재가 아쉬움.

### C. 워크플로우 단절
- **Vim 사용자**: 마우스 없이는 아티팩트 내비게이션 불가능.
- **속도**: 레이턴시가 일부 사용자의 "몰입(flow)"을 방해함.
- **멀티모달**: PM들은 텍스트 묘사 대신 스크린샷 업로드를 원함.

## 3. 핵심 페인 포인트 (Top Pain Points)
1.  **"내 코드를 모른다"**: import나 문서를 자동으로 가져와서 참고하지 않음.
2.  **"설정이 어렵다"**: 수동 JSON 편집은 구시대적임.
3.  **"내 디자인을 무시한다"**: 커스텀 토큰 대신 일반적인 Tailwind 클래스를 사용함.

## 4. 전략적 기회 (Strategic Opportunities)
- **스마트 컨텍스트 엔진**: import를 자동으로 파싱하고 `README.md`/`docs/`를 읽어 에이전트가 상황을 인식하게 함.
- **멀티모달 지원**: UI 생성을 위한 이미지 업로드 기능 허용.
- **로컬 모델 지원**: 오프라인/프라이버시 중심의 코딩을 위해 Ollama 등의 통합 지원.
